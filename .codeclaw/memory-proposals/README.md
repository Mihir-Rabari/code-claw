# Memory Proposals

Reviewable memory change packages live here.

## Purpose

A memory proposal is the only path from observation to durable memory. It is intentionally PR-shaped so a human can review it before merge.

## Workflow

1. Collect relevant observations.
2. Draft the proposed memory change.
3. Open or update a PR for human review.
4. Record review outcome.
5. Merge accepted parts into `REALITY.md`, `concepts/`, `debates/`, or `decisions/`.

## Suggested filename

`YYYY-MM-DD-short-slug.md`

## Suggested template

```md
# Memory Proposal: <short title>

- **Status:** draft | in-review | accepted | changes-requested | rejected
- **Date:** YYYY-MM-DD
- **Related observations:** <links>
- **PR:** <link or placeholder>

## Summary

<what memory change is being proposed>

## Proposal

- <what to add, update, or promote>

## Why this should be remembered

- <reason 1>
- <reason 2>

## Review notes

- **Human reviewer:** <name>
- **Outcome:** <pending / approved / rejected / revised>
- **Comments:** <review summary>
```
