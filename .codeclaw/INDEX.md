# INDEX.md

Start here when navigating repository memory.

## Purpose

This directory is the repository-owned memory layer for CodeClaw. It is markdown-first, auditable, and intentionally human-reviewed.

## Map

- `CLAW.md` — top-level memory policy and lifecycle
- `REALITY.md` — current best-known facts about the repo
- `MEMORY_GUIDE.md` — how to write and promote memory
- `observations/` — atomic evidence notes
- `memory-proposals/` — PR-ready proposal bundles
- `concepts/` — distilled reusable understanding
- `debates/` — unresolved questions and tradeoffs
- `decisions/` — accepted commitments and rules

## Working rule

Write new knowledge in the narrowest file that fits:

- evidence goes in `observations/`
- reviewable change packages go in `memory-proposals/`
- durable truths go in `REALITY.md`
- reusable abstractions go in `concepts/`
- unresolved choices go in `debates/`
- accepted commitments go in `decisions/`

## Reading order

1. `CLAW.md`
2. `REALITY.md`
3. relevant folder README/template
4. the specific observation, proposal, concept, debate, or decision you need

## No chronicle rule

Do not treat this directory like a story log. Keep entries atomic and sortable by purpose, not by narrative.
