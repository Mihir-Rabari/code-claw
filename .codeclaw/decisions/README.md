# Decisions

Accepted commitments live here.

## Rules

- Decisions are the durable outcome of human review.
- Keep decisions crisp and enforceable.
- Link back to the proposal or debate that justified the decision.
- If a decision changes, do not erase the old one silently; add a new decision or correction note.

## Suggested filename

`short-slug.md`

## Suggested template

```md
# Decision: <name>

- **Status:** accepted
- **Date:** YYYY-MM-DD
- **Source:** <proposal, debate, or review link>

## Decision

<what the repo has committed to>

## Implications

- <what this changes>

## Notes

- <optional constraints or exceptions>
```
